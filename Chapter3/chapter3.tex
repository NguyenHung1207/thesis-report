\section{Tổng quan hệ thống}

Hệ thống được xây dựng là một nền tảng web hỗ trợ kiểm thử tự động ứng dụng Web. Mục tiêu chính của hệ thống là giúp người dùng tạo, quản lý và thực thi các ca kiểm thử cho website mục tiêu mà không cần trực tiếp viết mã kiểm thử phức tạp. Thay vì yêu cầu người dùng tự viết script bằng các framework kiểm thử truyền thống, hệ thống cho phép người dùng mô tả mục tiêu kiểm thử bằng ngôn ngữ tự nhiên, sau đó sử dụng AI agent kết hợp với browser automation để thực hiện kiểm thử trên trình duyệt.

Về mặt sử dụng, hệ thống hỗ trợ người dùng tạo project kiểm thử, khai báo địa chỉ website cần kiểm thử, scan/crawl website để ghi nhận các trang có thể truy cập, tạo test case từ prompt, sinh dữ liệu kiểm thử, chạy test case bằng agent, lưu lại lịch sử thực thi, sinh replay script và chạy lại các kịch bản kiểm thử đã ghi nhận. Sau mỗi lần chạy, hệ thống lưu bằng chứng thực thi như log, screenshot, action history và tổng hợp thành báo cáo để người dùng có thể quan sát, đánh giá và phân tích nguyên nhân khi ca kiểm thử thất bại.

Người dùng có thể bắt đầu từ một mục tiêu kiểm thử ở mức nghiệp vụ, sau đó hệ thống hỗ trợ chuyển mục tiêu này thành test case, thực thi trên trình duyệt và ghi nhận kết quả để tái sử dụng trong các lần kiểm thử sau.

\section{Nhóm người dùng}

Hệ thống hiện tại tập trung vào một nhóm người dùng chính là \textbf{người dùng hệ thống}. Nhóm người dùng này có thể là QA, tester, developer hoặc người dùng nghiệp vụ có nhu cầu tạo và chạy kiểm thử tự động cho website. Hệ thống không phân tách vai trò quản trị viên riêng trong phạm vi hiện tại, do đó các chức năng chính được thiết kế xoay quanh người dùng trực tiếp thao tác với project và test case.

Người dùng hệ thống có thể thực hiện các thao tác chính sau:

\begin{itemize}
    \item Đăng ký, đăng nhập và đăng xuất khỏi hệ thống.
    \item Tạo và quản lý project kiểm thử.
    \item Khai báo website mục tiêu thông qua base URL.
    \item Scan/crawl website để ghi nhận các trang và luồng điều hướng có thể truy cập.
    \item Tạo test case từ mô tả bằng ngôn ngữ tự nhiên.
    \item Sinh và quản lý dữ liệu kiểm thử.
    \item Chạy test case bằng AI agent hoặc replay script.
    \item Chạy test case với nhiều bộ dữ liệu.
    \item Quản lý test suite để gom nhóm nhiều test case và chạy theo lô.
    \item Quản lý object repository để tái sử dụng thông tin định vị phần tử.
    \item Sửa step thông qua fast-forward inspect và hỗ trợ AI.
    \item Xem log, screenshot, evidence, báo cáo và phân tích nguyên nhân thất bại.
\end{itemize}

Việc tập trung vào một nhóm người dùng chính giúp hệ thống đơn giản hơn về mặt phân quyền trong giai đoạn hiện tại. Thay vì xây dựng nhiều vai trò phức tạp, hệ thống ưu tiên hoàn thiện luồng nghiệp vụ cốt lõi của kiểm thử tự động: tạo project, tạo test case, chạy test, ghi nhận kết quả, phát lại kịch bản và hỗ trợ sửa lỗi.

Nhìn chung, nhóm yêu cầu bắt buộc tập trung vào luồng kiểm thử cốt lõi: quản lý project, tạo test case, chạy test bằng agent, sinh replay script, chạy replay và xem báo cáo. Các yêu cầu như test data, test suite, object repository, self-healing và fast-forward inspect đóng vai trò mở rộng, giúp hệ thống phù hợp hơn với kiểm thử hồi quy và bảo trì kịch bản kiểm thử trong thực tế.

\section{Kết chương}

Chương này đã trình bày phân tích yêu cầu của hệ thống kiểm thử tự động ứng dụng Web. Nội dung chương bao gồm tổng quan hệ thống, nhóm người dùng, yêu cầu chức năng, yêu cầu phi chức năng, sơ đồ use case và đặc tả rút gọn các use case chính. Qua đó, chương làm rõ phạm vi chức năng mà hệ thống cần đáp ứng, từ đăng ký và đăng nhập, quản lý project, scan/crawl website, tạo test case từ prompt, chạy test bằng agent, sinh replay script, chạy với dataset, quản lý object repository, đến phân tích lỗi và báo cáo kết quả.

Các yêu cầu được phân tích trong chương này là cơ sở cho việc thiết kế kiến trúc và giải pháp kỹ thuật ở Chương~\ref{Chapter4}. Đặc biệt, hệ thống được định hướng không chỉ là công cụ chạy script kiểm thử, mà là một nền tảng hỗ trợ toàn bộ quy trình kiểm thử tự động ứng dụng Web, kết hợp giữa giao diện quản lý, backend lưu trữ, agent worker, Browser-use, Playwright và mô hình ngôn ngữ lớn.
